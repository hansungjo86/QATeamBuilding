# QA팀 업무 프로세스

**문서 유형:** 팀 프로세스 정의서  
**적용 대상:** QA / 기획 / 개발(클라+서버) / 아트  
**기준 도구:** Jira / Confluence / Google Sheets / TC 자동화 파이프라인

---

## 1. 전체 SDLC 흐름

### [기획] 단계

1. **[기획]** 기획서 작성 (Confluence)
2. **[기획]** Jira [기획] 일감 → "완료"
3. **[기획]** 기획 완료 알림 → 개발팀([클라]+[서버]+[아트]) 및 [PD]([PM]+[QA])
   - **[QA]** 기획서 리뷰 → 기획서 내 문의사항 또는 잠재적 리스크 체크
     - 이슈 있음 → 기획자에게 문의 후 해소
     - 이슈 없음 → TC 자동화 파이프라인 실행 → TC 작성 진행

### [개발] 단계

4. **[개발]** Jira [클라] + [서버] 일감 → 개발 시작
5. **[개발]** 개발 완료 → Jira [클라]/[서버] → "테스트"
   - **[QA]** TC 작성 완료 → 기획자에게 TC 전달

### [QA] 단계

6. **[기획]** 기획자 테스트 시작
7. **[PM]** Jira [QA] 일감 생성
8. **[QA]** TC 실행 및 결과 기록
9. **[QA]** 이슈 없음 → Jira [QA] → "완료"
10. **[QA]** 이슈 있음 → 버그 Jira 생성 → 개발팀 수정 → 재테스트

---

## 2. 단계별 상세 프로세스

### 2-1. 기획 단계

**담당:** 기획자

| 순서 | 작업 | 도구 | 산출물 |
|------|------|------|--------|
| 1 | 기획서 작성 | Confluence | 기획 문서 |
| 2 | 기획 완료 후 Jira [기획] 일감 → "완료" 상태 변경 | Jira | 상태 변경 |
| 3 | 개발팀([클라]+[서버])에 개발 시작 알림 | Jira 코멘트 또는 알림 | - |

**QA팀 액션:**
> 기획자가 [기획] 일감을 "완료"로 변경하는 시점에, QA팀은 **TC 사전 준비**를 시작할 수 있습니다.  
> Confluence 기획서 링크를 확보하여 TC 자동화 파이프라인을 미리 실행해두면 개발 완료 시점에 바로 테스트 진입이 가능합니다.

---

### 2-2. 개발 단계

**담당:** 개발자 (클라이언트 + 서버)

| 순서 | 작업 | 도구 |
|------|------|------|
| 1 | [기획] 일감 "완료" 확인 후 개발 시작 | Jira |
| 2 | 개발 완료 후 Jira [클라] 또는 [서버] 일감 → "테스트" 상태 변경 | Jira |

**QA팀 액션:**
> [클라] 또는 [서버] 일감이 "테스트" 상태로 변경되면 QA팀은 테스트 진입 가능 여부를 확인합니다.

---

### 2-3. QA 단계 ← 핵심

**담당:** QA팀

[클라]/[서버] 일감이 "테스트"로 전환되면 **[QA] 일감이 자동 생성**되고, 기획자 테스트와 동시에 QA가 시작됩니다.

#### Step 1. TC 자동화 파이프라인 실행

Confluence 기획서 링크를 기반으로 TC를 자동 생성합니다.

```
/tc-v2  <Google Sheets URL>  <Confluence 기획서 URL>
```

| 항목 | 내용 |
|------|------|
| 입력 | Confluence 기획서 URL |
| 소요 시간 | 약 40분 |
| 결과물 | Google Sheets — TC 자동 작성 완료 |

> **기획 변경이 있었던 경우:**
> ```
> "기획 변경됐어" + Confluence 변경된 기획서 URL
> ```
> 기존 TC에서 변경된 부분만 자동 갱신됩니다.

#### Step 2. TC 검토

파이프라인이 완료되면 QA팀원은 생성된 TC를 확인합니다.

**확인 항목:**
- 기획서의 주요 기능이 TC에 빠짐없이 반영되었는가
- 검증단계(정상/부정/예외) 비율이 적절한가
- 취소선 처리된 기능이 TC에서 제외되었는가
- 추후 구현 항목이 비고란에 표기되었는가

#### Step 3. TC 실행 및 결과 기록

Google Sheets TC를 기준으로 테스트를 실행하고 결과를 기록합니다.

| 결과값 | 의미 |
|--------|------|
| PASS | 정상 동작 확인 |
| FAIL | 버그 발견 |
| BLOCK | 선행 조건 미충족으로 테스트 불가 |
| 미진행 | 아직 테스트하지 않음 |
| N/A | 해당 플랫폼 미해당 |

#### Step 4. 이슈 처리

**FAIL / BLOCK 발생 시:**

```
Google Sheets에 결과 기록
    ↓
Jira 버그 일감 생성
    ↓
담당 개발자([클라] 또는 [서버])에 할당
    ↓
개발자 수정 완료 → Jira 버그 일감 "완료"
    ↓
QA팀 재테스트 → 결과 업데이트
```

#### Step 5. QA 완료 처리

모든 TC의 결과가 PASS / N/A로 확인되면:

- Jira [QA] 일감 → **"완료"** 상태 변경
- Google Sheets TC 링크를 Jira 코멘트에 첨부
- 대시보드 자동 갱신 확인

---

## 3. Jira 일감 상태 흐름

```
[기획] 진행중 → 완료
[서버] 시작 전 → 진행중 → 테스트
[클라] 시작 전 → 진행중 → 테스트
[QA]  (자동 생성) → 진행중 → 완료
```

### 직군별 상태 변경 책임

| 일감 | 상태 변경 주체 | 변경 시점 |
|------|-------------|---------|
| [기획] → 완료 | 기획자 | 기획서 작성 완료 시 |
| [클라] → 테스트 | 클라이언트 개발자 | 클라 개발 완료 시 |
| [서버] → 테스트 | 서버 개발자 | 서버 개발 완료 시 |
| [QA] → 진행중 | QA팀원 | TC 실행 시작 시 |
| [QA] → 완료 | QA팀원 | 전체 TC PASS 확인 시 |

---

## 4. 직군별 역할과 책임

### 기획자

| 책임 | 내용 |
|------|------|
| 기획서 작성 | Confluence에 작성, QA팀이 파이프라인에 입력할 수 있는 형태로 유지 |
| Jira 상태 관리 | [기획] 일감 완료 처리 |
| 기획 변경 시 | 변경 내용을 QA팀에 즉시 공유 (Confluence 업데이트 + Jira 코멘트) |
| 기획자 테스트 | [클라]/[서버] "테스트" 전환 후 기본 동작 확인 |

### 개발자 (클라 + 서버)

| 책임 | 내용 |
|------|------|
| 개발 완료 처리 | [클라]/[서버] 일감 → "테스트" 상태 변경 |
| 버그 수정 | QA팀이 생성한 버그 Jira 대응 |
| 재배포 알림 | 수정 후 QA팀에 재테스트 요청 |

### QA팀원

| 책임 | 내용 |
|------|------|
| TC 생성 | [QA] 일감 생성 시점에 파이프라인 실행 |
| TC 검토 | 자동 생성된 TC 품질 확인 |
| TC 실행 | Google Sheets 기준으로 테스트 수행 및 결과 기록 |
| 버그 리포트 | FAIL 발생 시 Jira 버그 일감 생성 및 개발팀 할당 |
| 완료 처리 | 전체 PASS 확인 후 [QA] 일감 완료 처리 |

---

## 5. 기획 변경 발생 시 프로세스

개발 중 또는 QA 중 기획이 변경되는 경우:

```
기획자: Confluence 기획서 업데이트
    ↓
기획자: 해당 Jira 일감 코멘트에 변경 내용 기재
    ↓
QA팀: TC 갱신 명령어 실행
    ("기획 변경됐어" + 변경된 Confluence URL)
    ↓
시스템: 변경된 부분만 자동 갱신 + 영향받는 TC 결과 초기화
    ↓
QA팀: 변경된 TC 재테스트
```

> ⚠️ 기획 변경 시 반드시 QA팀에 즉시 공유해야 합니다.  
> 공유 없이 기획이 변경되면 TC와 실제 기능이 불일치하게 됩니다.

---

## 6. 완료 기준 (Definition of Done)

| 항목 | 기준 |
|------|------|
| TC 전체 결과 | PASS 또는 N/A |
| 버그 Jira | 전부 "완료" 상태 |
| Google Sheets | 결과 기록 완료, 대시보드 반영 |
| Jira [QA] | "완료" 상태 변경 |

---

## 7. 참고 사항

- TC는 Google Sheets에 자동 생성됩니다. 직접 수정하지 마세요.
- 기획서 형식(Confluence / PDF / Word / Excel) 모두 지원됩니다.
- 여러 기능을 동시에 처리할 경우 하나가 실패해도 나머지는 계속 진행됩니다.
- 파이프라인이 중단되면 동일 명령어로 재실행 시 중단 지점부터 재개됩니다.
