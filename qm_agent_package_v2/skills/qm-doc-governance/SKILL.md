# QM 문서 거버넌스 스킬

## 핵심 원칙 (4가지 작성 지침)

본 프로젝트의 모든 표준 문서는 다음을 준수합니다.

1. **한글화**: 본문 한글, 영문 약어·기술용어만 영문 유지 (TC, KPI, CAPA, QMS 등)
2. **출처 명시**: 외부 정보 인용 시 URL 명시 (각 문서 하단 출처 섹션)
3. **축약·의역 검증**: 영문 원문 인용 시 영문 + 한국어 의역 표시 (`*원문 인용*` / `*한국어 의역*`)
4. **할루시네이션 방지**: 외부 표준 출처가 없는 부분은 "권고안" 표시

## 문서 일관성 검증 체크리스트

### A. 용어 일관성

| 점검 항목 | 통과 기준 |
|---------|--------|
| 동일 개념의 용어 통일 | 여러 문서에서 같은 단어 사용 |
| 약어 풀이 첫 등장 명시 | "QM(Quality Management)" 형식 |
| 외래어 표기법 준수 | 한국어 어문 규범 |

### B. 버전 일관성

- 모든 표준 문서에 버전 표기 필수
- 상호 참조 시 같은 버전 가리키기
- 버전 이력 표 갱신 확인

### C. 상호 참조 정확성

- URL 유효성 (404 없음)
- Confluence 페이지 ID 살아있음
- GitHub 링크 살아있음

### D. 외부 출처 인용 규칙

```
> 출처: [출처명] - [페이지 제목]
> URL: [전체 URL]
> *영문 원문 인용 (그대로): "[영문 원문]"*
> *한국어 의역 (의미 손실 없음): [한국어 번역]*
```

## 변경 이력 작성 형식

```
## 변경 이력 (vX.Y → vX.Y+1)

**변경 일시**: YYYY-MM-DD
**작성자**: [이름]
**검토자**: [이름]

### 변경 내역

- [문서명] L23: "이슈" → "버그"
- [문서명] L45: 버전 표기 추가
- [문서명] 5장: 신규 KPI 정의 추가

### 변경 사유

[다중 문서 일관성 확보 / 외부 표준 변경 반영 / 사용자 요청 등]

### 영향 분석

- 영향받는 문서: [목록]
- 후속 조치 필요: [있다면 명시]
```

## 표준 문서 분류 (사용자 환경)

| 문서 유형 | 위치 | 주된 검증 포인트 |
|---------|------|---------------|
| QA 팀 업무 프로세스 | 사용자 GitHub | SDLC 흐름, 직군별 R&R |
| QA 파트 규약 | 내부 (Confluence) | 직급별 R&R, 협업 원칙 |
| KPI 정의서 | 내부 | 측정 기준, 계산식 |
| 릴리즈 체크리스트 | 내부 | 항목 정의, DoD |
| TC 작성 표준 | TC Team v2 skills | 형식, 검증 비율 |
| QM 에이전트 가이드 | 본 패키지 | 호환성, 운영 원칙 |

## 권고안 vs 표준의 구분 표시

본 프로젝트의 문서에는 두 종류가 혼재합니다.

### 표준 (Standard)
외부 검증 가능한 출처가 있는 사실. 인용 표시 필수.

```
> 출처: ISO 9001:2015 9.2절 (외부 표준)
*원문 인용 → 한국어 의역*
```

### 권고안 (Recommendation)
사용자 환경에 맞춘 운영 권고. 표준이 아님을 명시.

```
> 본 표·체크리스트는 외부 표준이 아닌 운영 권고안입니다.
> 사용자 환경·팀 합의에 따라 조정 필요.
```

## ISO 9001:2015 7.5 Documented Information 핵심 요구사항

> 출처: ISO 9001:2015 7.5절 (일반론)

QMS의 문서화된 정보는 다음을 포함해야 합니다.

- 표준이 요구하는 문서화된 정보
- 조직이 QMS 효과성을 위해 필요하다고 판단한 문서화된 정보

문서 통제는 다음을 보장해야 합니다.

- 적절한 식별·기술 (제목, 날짜, 작성자, 참조 번호)
- 적절한 형식·매체
- 적합성·유효성 검토·승인
- 변경 통제
- 보관·접근·검색·사용
- 폐기 통제

본 권고는 ISO 9001 인증을 받지 않은 조직에도 좋은 운영 가이드가 됩니다.
