# QM KPI 스킬

## KPI 카테고리 (3분류)

### A. 품질 지표 (Quality Metrics)

품질 그 자체에 대한 측정.

| KPI | 정의 | 계산식 | 데이터 소스 | 권장 주기 |
|-----|------|------|----------|--------|
| TC PASS율 | 전체 TC 중 PASS의 비율 | PASS / 전체 × 100 | Google Sheets | 주간 |
| TC FAIL율 | 전체 TC 중 FAIL의 비율 | FAIL / 전체 × 100 | Google Sheets | 주간 |
| TC BLOCK율 | 선행 조건 미충족 비율 | BLOCK / 전체 × 100 | Google Sheets | 주간 |
| 회귀 발생률 | 이전 PASS → FAIL 전환 비율 | 회귀 발생 / 전체 × 100 | 시계열 비교 | 스프린트 |
| 핫픽스 횟수 | 라이브 후 긴급 수정 횟수 | 카운트 | Jira (Hotfix 라벨) | 월간 |
| 평균 버그 해결 시간 (MTTR) | 일감 생성 → 완료까지 평균 | (완료시간 − 생성시간) 평균 | Jira | 월간 |

### B. 효율성 지표 (Efficiency Metrics)

QA 활동의 효율을 측정.

| KPI | 정의 | 계산식 | 권장 주기 |
|-----|------|------|--------|
| TC 자동 생성 적중률 | 인간 수정 없이 채택된 자동 TC 비율 | 무수정 채택 / 자동 생성 총수 × 100 | 월간 |
| LLM 출력 검증 시간 | 자동 생성 TC 검증 평균 시간 | 검증 시간 합 / 검증 TC 수 | 주간 |
| 자동화 ROI | 시간 절감 효과 | (수동 추정 − 자동+검증) / 수동 추정 × 100 | 월간 |
| 평균 회귀 사이클 시간 | 빌드 → 결과 보고까지 시간 | TC Team v2 파이프라인 로그 | 스프린트 |

### C. 프로세스 지표 (Process Metrics)

프로세스 준수도를 측정.

| KPI | 정의 | 데이터 소스 |
|-----|------|----------|
| 일감 상태 흐름 준수율 | 정의된 흐름을 따른 일감 비율 | Jira |
| TC 누락 발견율 | 사후 발견된 TC 누락 비율 | 회고록 + Confluence |
| 기획 변경 처리 시간 | 변경 → TC 갱신까지 시간 | Confluence + TC Team v2 로그 |

## 자동화 ROI 계산 (핵심)

> 출처: 삼성SDS 인사이트리포트 - 생성형 AI의 테스트 자동화 활용 (https://www.samsungsds.com/kr/insights/generative-ai-for-test-automation.html)
> *원문 인용: "테스터가 테스트를 실행하기보다 LLM 결과를 정제하는 데 상당한 시간을 소비하는 상쇄 효과에 직면한다."*

### 계산 절차

1. **수동 작성 추정 시간 (T_수동)**
   - 자동화 도입 전 데이터가 있다면 그 평균
   - 없다면 동일 복잡도 기획서로 1회 수동 작성하여 측정

2. **자동 생성 + 검증 실제 시간 (T_자동)**
   - TC Team v2 파이프라인 실행 시간
   - 인간 검증·수정 시간

3. **ROI 계산**
   ```
   ROI = (T_수동 − T_자동) / T_수동 × 100
   ```

### 해석

| ROI 값 | 의미 | 권고 행동 |
|-------|------|---------|
| 50% 이상 | 매우 효율적 | 유지 + 다른 영역 확장 |
| 20%~50% | 효율적 | 유지 |
| 0%~20% | 미미한 효율 | 개선 모니터링 |
| 0% 이하 | 비효율 | 즉시 재설계 또는 자동화 제외 |

## 추세 분석 원칙

### 1. 최소 2개 기간 비교

- 단일 기간 수치만 가지고 좋고/나쁨 단정 금지
- 비교 기간 데이터 없으면 "추세 판단 불가" 명시

### 2. 이상 신호 정의 (운영 권고안)

> 다음 임계값은 일반 권고이며, 팀 합의로 조정 필요 (할루시네이션 방지 명시).

| 신호 | 임계값 (권고) | 권고 행동 |
|------|----------|---------|
| TC FAIL율 급증 | 직전 대비 +5%p 이상 | 회고에서 원인 분석 |
| 회귀 발생률 증가 | 직전 대비 +3%p 이상 | 회귀 테스트 강화 검토 |
| 자동화 ROI 하락 | 직전 대비 -10%p 이상 | 프롬프트 재설계 검토 |
| MTTR 증가 | 직전 대비 +20% 이상 | 개발팀 협업 점검 |

### 3. 원인 단정 금지

KPI 악화 시 "원인은 X다" 단정 금지. 가능한 요인을 우선순위별로 나열.
